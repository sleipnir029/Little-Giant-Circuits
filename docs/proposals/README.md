# Proposals

This directory holds proposal documents for changes to `CLAUDE.md`.

## Why This Exists

`CLAUDE.md` is the operating manual for this project. It must not be rewritten casually during implementation. To protect it, any rule change goes through a proposal review cycle before touching the file.

This is defined in `CLAUDE.md §"Updating This File (Proposal Mode)"` and `PROJECT_PLAN.md §11`.

## How to Submit a Proposal

1. Create a file named `claude_update_<YYYY-MM-DD>.md` in this directory.
2. The file must answer these four questions:
   - **Current issue:** what is wrong or missing in the current rule?
   - **Proposed change:** what exactly should change in `CLAUDE.md`?
   - **Benefit:** why is this change worth making?
   - **Drift risk:** could this change weaken the learning-first constraint or other non-negotiable principles?
3. Do not edit `CLAUDE.md` yet. Wait for human review.
4. After human approval is confirmed, make the edit and delete or archive the proposal.

## What Does Not Need a Proposal

- Typo fixes or broken-link corrections in `CLAUDE.md`.
- Adding content to `docs/` that does not change a rule.
- Creating new phase docs or open questions.
